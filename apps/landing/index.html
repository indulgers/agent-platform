FEATURES
功能

The agent runtime, minus the lock-in. 一套 agent 运行时，没有任何 lock-in。

Three pieces do the heavy lifting: a provider layer that talks to any major LLM, a tool registry that runs your code, and an SSE pipeline that streams everything back. Replace any piece without rewriting the rest. 三件事撑起整个系统：能对接任意主流 LLM 的 provider 层、跑你自己代码的工具注册表、把整个流程实时回传的 SSE 通道。任何一块都可以替换，不会牵动其它部分。

Multi-tool plan / act loop 多工具 plan / act 循环

The runner streams tokens until the model emits a tool call, runs the tool, feeds the result back, repeats. Max-iterations guard. Tools are zod-typed, validated at the boundary. runner 边流式回吐 token 边监听 tool call，调用、回填、继续。带最大迭代次数兜底。工具用 zod 定义，边界处自动校验。

http_fetch vector_search + your own

SSE streaming, end to end 端到端 SSE 流

One typed event union — token · tool_call · tool_result · done — used by both server and client. No invented adapters, no buffered chunks. 一套带类型的事件 union — token · tool_call · tool_result · done — 前后端共用。不发明 adapter，不缓冲整段。

SseEvent @agent-platform/shared

Plug any LLM provider 任意 LLM provider 一键接入

OpenAI, Anthropic, and DeepSeek ship in the box. Anything OpenAI-protocol-compatible (Together, Groq, vLLM, Ollama) is a 10-line subclass. OpenAI、Anthropic、DeepSeek 开箱即用。任何 OpenAI 协议兼容的服务（Together、Groq、vLLM、Ollama）只需要一个 10 行的子类。

openai anthropic deepseek + compatible
HOW IT WORKS
怎么运作

Three moving parts. One typed event stream. 三件事，一条带类型的事件流。

Same loop every request: you ask, the runner plans and runs tools, tokens stream back. Everything you see in the UI corresponds to a typed SseEvent the server actually emitted — no synthesised animations. 每次请求都是同一个循环：你提问、runner 规划并调用工具、token 流式回传。前端看到的每一帧都对应服务器真实发出的 SseEvent —— 没有 fake 动画。

01 / ASK

POST a message 提一个问题

Client POSTs to /api/agents/:id/messages with the user's text and an Authorization header. JWT-guarded. 客户端 POST 到 /api/agents/:id/messages，带上用户文本和鉴权头。受 JWT 保护。 

POST /api/agents/abc/messages
{
  "content": "Summarise weekly metrics"
}
02 / PLAN + ACT

Runner loops until done runner 循环直到收尾

The runner streams tokens. On a tool_call it pauses, validates the args with zod, runs the tool, feeds the result back, continues. Capped at AGENT_MAX_ITERATIONS. runner 边流式输出 token，碰到 tool_call 就暂停、用 zod 校验参数、运行工具、把结果填回、继续。最多 AGENT_MAX_ITERATIONS 步兜底。 

while (iter++ < MAX) {
  const r = await provider.stream(msgs)
  if (!r.toolCalls.length) break
  msgs.push(...await runTools(r))
}
03 / STREAM

SSE all the way down SSE 一路打到前端

Every step emits a typed event on the SSE channel. The browser parses token / tool_call / tool_result / done and updates the UI as they arrive. 每一步在 SSE 通道上发出一条带类型的事件。浏览器解析 token / tool_call / tool_result / done 并实时更新 UI。 

data: {"type":"token","delta":"Active"}
data: {"type":"tool_call","name":"http_fetch"}
data: {"type":"done"}
USE CASES
使用场景

Built for the chores you don't enjoy. 替你跑那些不想干的活。

01 · Research assistant

Pull, read, summarise. 拉网页、读完、总结好。

Ask for a competitor's pricing or a paper's findings. The agent fetches, parses, and writes you a 3-bullet summary with citations. 问竞品定价、问一篇论文的结论。agent 自己抓取、解析、写出带引用的 3 条要点。

http_fetch vector_search
02 · Data ops

Slice your metrics in a sentence. 一句话切你的数据。

Point it at your dashboard, your warehouse, or a CSV. Ask "WoW change in p90 latency by endpoint". Get the answer, the SQL it ran, and a chart-ready table. 让它对着你的看板 / 数据仓库 / CSV。问 "按 endpoint 看 p90 延迟周环比"。回你答案、它跑过的 SQL、以及可以直接画图的表。

sql_query http_fetch
03 · Workflow automation

Long jobs without the babysitting. 长任务，不用守着。

Heavy work (crawls, embedding batches, exports) hands off to BullMQ. You get a taskId, you watch it, you come back to results. 重活（爬虫、批量 embedding、导出）下发给 BullMQ。你拿到 taskId，去看进度，结果好了回来取。

BullMQ + your tool
